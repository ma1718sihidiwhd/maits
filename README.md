---

## 🔷 Approach

- 定義されていない業務は品質が安定しない  
- 評価基準は事前に明確であるべき  
- 業務は第三者が再現可能である必要がある  

上記を前提に、

- 要件の曖昧さを構造化し  
- 評価観点を定義し  
- プロセスおよび役割（RACI）を整理し  
- 実運用に適用可能な形へ落とし込む  

ことを重視しています。
